    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-key-manager" class="anchor" aria-hidden="true" href="#key-manager"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Key Manager</h1>
<h1></h1>
<h1><a id="user-content-background" class="anchor" aria-hidden="true" href="#background"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Background</h1>
<p>In the MOSIP platform, there are many types of keys used in various modules. The users might book an appointment using the Pre-Registration module. During this the users uses the public key to encrypt the message and Pre-Registration module uses the private key to decrypt the message. When these packets are pulled by the Registration module, the Registration module can use the public key of Pre-Registration module to establish the connection.</p>
<p>The keys should be maintained and managed as a separate entity apart from the actual modules. These keys are rotated based on a specific security policy.</p>
<p><a target="_blank" rel="noopener noreferrer" href="https://github.com/mosip/mosip/blob/master/design/kernel/_images/kernel_keymanager_connections_diagram.jpg"><img src="https://github.com/mosip/mosip/raw/master/design/kernel/_images/kernel_keymanager_connections_diagram.jpg" alt="ScreenShot" style="max-width:100%;"></a></p>
<h1><a id="user-content-solution" class="anchor" aria-hidden="true" href="#solution"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Solution</h1>
<p>The key solution considerations are</p>
<ul>
<li>--Following are the major components in the Key Manager,
<ol>
<li>Key generation algorithms.</li>
<li>Key rotation module, which will generate the new keys and map to the application for a specific timeframe.</li>
<li>A RESTful micro service, which will serve the public keys of the requested application.</li>
<li>Validator module to check the authenticity of the caller.</li>
<li>Key store to store the keys.</li>
</ol>
</li>
</ul>
<p><a target="_blank" rel="noopener noreferrer" href="/mosip/mosip/blob/master/design/kernel/_images/kernel_keymanager_components_diagram.png"><img src="/mosip/mosip/raw/master/design/kernel/_images/kernel_keymanager_components_diagram.png" alt="Components Diagram" style="max-width:100%;"></a></p>
<ul>
<li>
<p>--The sequence of the key generations and the rotation of the keys are as follows,</p>
<ol>
<li>During the initial setup, the key rotation duration are configured.</li>
<li>Applications are associated to the key rotation configuration</li>
<li>The key expiration check is done during the request time.</li>
<li>The decryption of data happens inside the key manager module.</li>
<li>The public key is returned to the requester.</li>
</ol>
</li>
<li>
<p>--Sequence diagrams are as follows,</p>
</li>
</ul>
